# Соответствие техническому заданию

Документ сопоставляет ТЗ (сбор данных для RAG из открытых источников) с текущей реализацией и перечисляет недостающие элементы.

---

## 1. Цель и общие требования

| Требование | Статус |
|------------|--------|
| Только открытые источники, без обхода защиты/ToS | ✅ Соблюдается (CyberLeninka — публичный поиск, задержки в конфиге) |
| Мультиязычность (ru, en, fr, de, es, zh, ja, ko, ar, he, yi) | ⚠️ Частично: коды языков в API и langdetect; **поиск по языку** реализован только в источнике CyberLeninka (и то ограниченно) |
| Анонимность: задержки, ротация user-agent | ✅ Задержки в `config/settings.yaml`; User-Agent в запросах; прокси не реализованы |
| Вход: ключевые слова, даты, URL (опц.), языки | ✅ `CollectRequest`: keywords, date_from/date_to, urls, languages |
| Выход: JSON/CSV/NDJSON, метаданные, чанки 500–2000 | ✅ JSON в ответе API, NDJSON по `/tasks/{id}/ndjson`; чанки по токенам в настройках |

---

## 2. Задача 1: Наукометрические базы и авторефераты диссертаций

| Источник | Статус |
|----------|--------|
| **CyberLeninka** | ✅ Реализован (поиск, парсинг статей, PDF, чанки) |
| **CrossRef** | ✅ Реализован (REST API: поиск, метаданные, DOI, аннотация, опционально PDF по link) |
| OpenAlex | ❌ Нет |
| arXiv | ❌ Нет |
| PubMed | ❌ Нет |
| РГБ (ldiss.rsl.ru) | ❌ Нет |
| dslib.net | ❌ Нет |
| OATD | ❌ Нет |
| DART-Europe | ❌ Нет |
| CiNii Dissertations | ❌ Нет |

**Выход по ТЗ:** `{title, authors, date, doi/url, abstract, full_text_chunks[], language, source}` — **реализован** в модели `RAGDocument` и в ответе API.

**Фильтры:** ключевые слова и даты передаются в источники; языки — в запросе (реальное применение зависит от источника).

---

## 3. Задача 2: Общий поиск по интернету (SERP)

| Требование | Статус |
|------------|--------|
| Локализованный поиск (SERP), топ 10–50 | ❌ Источник типа `google_serp` / SerpAPI не реализован |
| Поля snippet, relevance_score | ⚠️ В модели можно добавить; источника для SERP нет |

**Реализация:** нужен новый источник (например, SerpAPI, Bing API или парсинг поисковой выдачи с соблюдением ToS), регистрация в `sources.registry`, вызов из `collector` при `task_type=web_search`.

---

## 4. Задача 3: Сбор с конкретных сайтов по URL

| Требование | Статус |
|------------|--------|
| Указание URL, извлечение текста/PDF | ✅ `task_type=target_site`, поле `urls` в запросе |
| Фильтры по ключевым словам в тексте, датам, языкам | ⚠️ Сейчас страница по URL просто парсится; фильтрация по ключевым словам/дате внутри страницы не делается |

**Реализация:** в `collector.py` для `target_site` вызывается `source.fetch_article(url)`; это поддерживается только у источников с методом `fetch_article` (сейчас — CyberLeninka). Для произвольного сайта нужен универсальный краулер/парсер (например, по списку URL с BeautifulSoup/Playwright) и опциональная фильтрация по ключевым словам в теле текста.

## 5. Критерии приёмки (тестовые сценарии)

| Тест | Статус |
|------|--------|
| **Тест 1:** 10 авторефератов по «диссертация климат» (ru/en), структура и чанки | ⚠️ Возможно только по CyberLeninka (статьи, не только авторефераты); остальные базы не подключены |
| **Тест 2:** Поиск «экономика РФ 2025», top-20, 3 языка | ❌ Нет источника общего поиска (SERP) |
| **Тест 3:** Сайт example.com — извлечь статьи/PDF | ⚠️ Только если источник умеет `fetch_article` по произвольному URL; универсального парсера нет |

**Качество данных:** очистка текста, дедупликация по хешу, валидация — реализованы. Устойчивость к сбоям: повторные попытки и cooldown при капче/блокировке есть.

---

## 6. Что сделано в коде (кратко)

- **API:** FastAPI, `POST /api/v1/collect`, `GET /api/v1/tasks/{id}`, NDJSON, список источников.
- **Оркестратор:** `collector.run_collection` — поиск по источникам или `target_site` по URL, дедупликация, cooldown, закрытие сессий источников.
- **Источники:** **CyberLeninka** (поиск + парсинг статей + PDF), **CrossRef** (REST API, метаданные, DOI, аннотация, PDF/Unpaywall, полные тексты из link, фильтр по языку и релевантности).
- **Обработка текста:** нормализация PDF, чанкование 500–2000 токенов (tiktoken), определение языка (langdetect + коррекции), обобщённые паттерны колонтитулов; подавление предупреждений PyMuPDF при разборе PDF.
- **Модель выхода:** RAGDocument с полями по ТЗ плюс relevance_score, file_access, crawling_timestamp.
- **Конфиг:** `config/settings.yaml` (задержки, чанки, языки, экспорт, порог релевантности, настройки по источникам).
- **БД:** PostgreSQL (опционально), таблицы tasks/documents.
